## 摘要
开放词汇的3D场景图方法通常分两阶段运行：先重建，再用视觉-语言模型丰富，导致探索期间图不可查询。我们认为这种顺序耦合是不必要的，并提出一种异步架构，其中轻量级在线建图与重量级语义优化并发运行。基于概率体素的骨干网络增量式地维持稳定的对象身份，后台VLM智能体逐步丰富图。该框架通过语义闭环检测解决重复对象轨迹，附加细粒度视觉属性，并推导对象间的空间关系。多目标帧调度器通过选择覆盖多个目标的少量信息帧来分摊VLM成本。生成的场景图在探索期间可查询，并随时间增长语义丰富度。我们的方法在语义分割（ScanNet，Replica）上匹配或超越现有开放词汇3D场景图方法，并在三个视觉定位基准（Sr3D+，Nr3D，ScanRefer）上以15.3至18.8的A@0.25超越先前最佳水平。项目页面：https://denizbickici.github.io/thinkgraphs/

## Abstract
Open-vocabulary 3D scene graph methods typically operate in two stages: first reconstruct, then enrich with vision-language models, leaving the graph unqueryable during exploration. We argue that this sequential coupling is unnecessary and propose an asynchronous architecture in which lightweight online mapping runs concurrently with heavyweight semantic refinement. A probabilistic voxel-based backbone maintains stable object identities incrementally, while background VLM agents progressively enrich the graph. This framework resolves duplicate object tracks through semantic loop closure, attaches fine-grained visual attributes and derives spatial relations between objects. A multi-target frame scheduler amortizes VLM cost by selecting a small set of informative frames that jointly cover multiple targets. The resulting scene graph is queryable during exploration and grows in semantic richness over time. Our method matches or outperforms existing open-vocabulary 3D scene graph methods on semantic segmentation (ScanNet, Replica) and surpasses the prior state-of-the-art across three visual grounding benchmarks (Sr3D+, Nr3D, ScanRefer) by 15.3 to 18.8 A@0.25. ## 1. 核心问题与整体含义（研究动机和背景）

现有开放词汇3D场景图方法通常采用两阶段串行流水线：**先完成完整的几何重建，再通过视觉-语言模型（VLM）进行语义丰富**。这意味着在探索（建图）过程中，场景图是不可查询的，无法支持实时交互任务（如机器人的在线视觉定位、增强现实中的语言查询）。另一类实时建图系统虽然支持查询，但仅能提供有限、封闭集的语义信息，牺牲了开放词汇的丰富性。

本文认为，这种顺序耦合是不必要的，其核心洞察是：**语义推理不需要等待稳定的图建成，它可以主动帮助稳定图**。作者提出一种异步架构 ThinkGraphs，让轻量级在线建图与重量级后台VLM推理并发运行，使场景图在探索期间始终保持可查询状态，且随时间积累越来越丰富的语义。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将在线建图（前端+后端）与异步VLM智能体解耦，通过概率体素投票机制保证对象标识的稳定性，同时通过后台代理（Critic Agent、Description Agent）增量式地修正和丰富场景图，不阻塞主循环。

### 关键技术细节

**前端：逐帧实例提取**
- 使用 Qwen3-VL-2B-Instruct 生成场景中的名词提示列表（上下文感知，比RAM++更准确）。
- 输入 Grounded-SAM v2（Grounding-DINO + SAM2），输出带标签和置信度的2D实例掩码。

**后端：3D关联与轨迹表示**（基于 OpenVox 概率体素，进行适配）
- **关联评分**：结合几何分数（体素一致性投票）和特征分数（CLIP文本标签余弦相似度）：
  - S(T_i, O) = λ_geo·S_geo + λ_feat·S_feat，其中 S_geo = (1/|V(O)|) Σ_v (c_{v,i} / c_v)，λ_geo=0.8，λ_feat=0.2，关联阈值 τ_assoc=0.4。
- **置信度加权标签投票**：每条新观测更新标签直方图 H_i(ℓ) += q（检测置信度），取最高分标签为共识标签 ℓ*_i。
- **文本引导的视觉特征选择**：从跟踪过程中积累的多个视角的CLIP视觉嵌入中，选与共识文本嵌入余弦相似度最高的作为代表特征 f*_i。同时使用自适应面积门控（面积 ≥ ρ_area × 历史最大面积）过滤低质量视角。
- **空间边**：根据轴对齐包围盒的几何启发式（上下左右等）确定，与房间中心锚点保持视角不变性。

**异步 VLM 智能体**
- **Critic Agent（语义闭环）**：检测并合并因增量关联漂移导致的重叠/碎片轨迹。
  - 候选对筛选：几何门（IoU_3D ≥ τ_iou=0.01，最少观测次数≥2）→语义门（SBERT余弦相似度 ≥ τ_sem=0.8）。
  - 候选对存入缓冲区，每 W_critic=50 帧刷新，选 top-K_critic=20 对提交给 VLM（GPT-5-mini）。
  - 使用 Set-of-Mark（SoM）标注图像，连同几何提示（包含率、偏移等），让VLM输出“MERGE”或“KEEP”决策。
  - 合并后统一点云、特征和标签直方图。
- **Description Agent（属性丰富）**：为每个节点添加材料、颜色、状态、纹理等细粒度属性。
  - **多目标帧调度器**：在滚动窗口 W_desc=30 帧内，贪心选择最多 K_desc=3 帧，使得选出的帧能覆盖最多未描述的目标。评分函数 U(f) = Σ_T w_T·ln(1+γ·Â_{T,f})，γ=100，背景类权重 w_T 降低。
  - 选定帧同样用 SoM 标记，发送给 VLM 一次调用即可同时丰富多个节点。返回的属性通过置信度加权投票累积到每个节点的属性直方图中。

## 6. 主要结论与发现

1. **异步架构有效**：ThinkGraphs 在语义分割和视觉定位上均**匹配或超越**所有基于批量重建的方法，证明“先重建再丰富”并非必要。
2. **语义闭环显著提升对象一致性**：Critic Agent 可修复增量关联中的碎片化和重复轨迹，将定位准确率提升0.8-1.5 A@0.25（Nr3D），且在大幅视点变化下依然有效。
3. **多目标帧调度大幅降低VLM开销**：平均每场景仅需26次VLM调用即完成全部节点的丰富，对比贪心逐帧方案降低了两个数量级。
4. **细粒度属性有效解决同类别歧义**：Description Agent 提供的材料和颜色等信息在Hard分项上带来3.5-7.1 A@0.25的提升。
5. **最佳结果**：
   - 语义分割：Replica上 mAcc 0.58, mIoU 0.37, f-mIoU 0.61；ScanNet上 mAcc 0.75, mIoU 0.44, f-mIoU 0.46。
   - 视觉定位：Sr3D+ A@0.25 43.0（超Open3DSG 15.3），Nr3D A@0.25 41.8（超18.8），ScanRefer A@0.25 52.9（超18.3）。

## 7. 优点

- **架构创新**：首次提出在线异步VLM智能体用于3D场景图，将“语义闭环”类比于几何SLAM中的闭环，思想新颖且实用。
- **高效性**：多目标帧调度 + 后台异步处理，使得昂贵VLM调用仅发生在关键时机，实际每场景VLM调用数 ~26次，具备实际部署潜力。
- **强性能**：在不依赖离线批处理情况下，达到并超过所有离线方法，验证了增量方案的有效性。
- **充分消融**：每个设计选择均有实验支撑，且通过不同评估器验证了结果鲁棒性。
- **可解释性**：提供Critic Agent行为的分类统计和错误案例分析，帮助理解系统行为。

## 8. 不足与局限

- **实时性不足**：前端模型（Qwen3-VL 2B + Grounded-SAM）单帧需要1.10秒，无法达到视频帧率（30fps），最直接的改进是替换为更高效的模型。
- **异步滞后**：VLM智能体相对于建图前缘有约25-30关键帧的延迟，导致快速探索中的早期物体可能尚未丰富，早期查询质量受限。
- **错误合并风险**：当大物体轴对齐包围盒过于膨胀时，可能错误地将小物体合并进去（如Fig.6第四例），需要更鲁棒的几何约束。
- **空间边仅使用启发式**：当前仅支持简单的方位关系，未使用VLM学习更复杂的功能关系（如“支撑”、“阻挡”），这与部分离线方法相比存在差距。
- **泛化局限**：仅在Replica和ScanNet这两套室内数据集上验证，未评估室外、动态或大规模场景。
- **计算资源**：直接依赖 GPT-5-mini 云API，无法完全离线，存在网络延迟和隐私风险。
