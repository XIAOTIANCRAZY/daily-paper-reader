## 摘要
LLM智能体越来越多地被用于以程序化方式将自然语言翻译成3D场景，但现有系统主要关注静态输出。仅从文本生成动态4D场景（其中液体流动、粒子发射、刚体级联、关节机构运动）仍 largely unexplored，尽管它们作为可编辑内容和视频生成及具身AI的基于物理的训练数据具有价值。两个挑战将动态情况与静态文本到场景的工作区分开来：智能体必须在一个连贯的场景中共同协调空间布局、多个物理求解器、时间序列、相机和光照；并且从渲染视频中验证运动正确性本质上比判断单个图像更难。我们提出SimWorlds：一个多智能体框架，用于从文本生成动态、可编辑的4D场景，具有Blender特定的程序化知识，一个规划者-编码者-审查者工作流程驱动固定有序的构建阶段序列，一个由确定性验证器强制执行的分层场景协议，以及一套捕获渲染图像无法揭示的机制故障的运行时状态检查工具。我们还引入了4DBuildBench，一个用于评估从文本提示生成的程序化动态3D场景的视觉保真度和物理一致性的基准。实验表明，SimWorlds优于先前的动态Blender生成基线。

## Abstract
LLM agents are increasingly used to translate natural language into 3D scenes in a procedural way, but existing systems focus on static output. Dynamic 4D scenes from text alone, in which liquids flow, particles emit, rigid bodies cascade, and articulated mechanisms move, remain largely unexplored despite their value as editable content and as physics-grounded training data for video generation and embodied AI. Two challenges set the dynamic case apart from static text-to-scene work: an agent must jointly coordinate spatial layout, multiple physics solvers, temporal sequencing, camera, and lighting in a single coherent scene, and verifying motion correctness from rendered video is fundamentally harder than judging a single image. We present SimWorlds: a multi-agent framework that produces dynamic, editable 4D scenes from text, with Blender-specific procedural knowledge, a planner-coder-reviewer workflow driving a fixed ordered sequence of construction stages, a layered scene protocol enforced by a deterministic verifier, and a runtime-state inspection tool suite that catches mechanism failures the rendered image cannot reveal. We also introduce 4DBuildBench, a benchmark for assessing both visual fidelity and physical consistency of the procedural dynamic 3D scenes generated from text prompts. Experiments show that SimWorlds outperforms prior dynamic Blender generation baselines.

## 1. 核心问题与整体含义（研究动机和背景）

**研究动机**：现有基于 LLM 智能体的文本转 3D 场景系统（如 SceneCraft、VIGA 等）主要生成静态场景，无法处理动态 4D 场景（如液体流动、粒子发射、刚体级联、布料飘动、关节机构运动等）。动态场景生成面临两大挑战：① 需要在单一连贯场景中联合协调空间布局、多个物理求解器、时间序列、相机和光照；② 从渲染视频中验证运动正确性比判断单张图像困难得多，且视觉上正确的渲染可能对应错误的底层机制（例如用关键帧动画伪造物理模拟）。此外，动态场景作为可编辑内容和视频生成/具身 AI 的物理训练数据具有重要价值，但目前缺乏端到端的解决方案。

**整体含义**：SimWorlds 提出了第一个能够从自然语言生成可编辑、基于物理模拟的动态 4D 场景（Blender .blend 文件）的多智能体系统，填补了该领域空白，可服务于下游图形编辑、仿真训练数据生成等任务。

## 2. 方法论：核心思想、关键技术细节与流程

**核心思想**：将动态 3D 场景生成视为**计划驱动的、机制感知的程序合成**。关键创新在于：不依赖视觉反馈（渲染图像）验证机制正确性，而是通过直接检查 Blender 运行时状态（修改器堆栈、物理缓存、动画通道等）来确保每种动态效果由正确的物理求解器实现，而非伪造。

**关键技术细节**：

- **多智能体架构**：包含三个角色：规划者（Planner）、编码者（Coder）、审查者（Reviewer），以及一个固定的构建阶段序列。
- **阶段化管线**：固定顺序的 8 个阶段：建模 → UV → 纹理 → 变形设置 → 运动 → 相机 → 光照 → 渲染。每阶段执行 `执行→验证→审查` 循环，采用有界重试机制（最多 10 次编码重试、5 次战术重规划）。
- **场景协议（Scene Protocol）**：在 Blender 原生场景图上叠加三层分层结构（L1 场景根、L2 系统分组、L3 逻辑对象），并显式声明表面接触、共动关系。协议由确定性验证器自动检查。
- **确定性验证器**：包含四组规则：结构规则、几何规则（BVH 距离测试）、软规则（警告）、计划-状态匹配规则（R1–R18 和 T 系列运动时序规则）。能捕捉漂浮物体、未声明穿透、不正确的父级链等渲染图像无法暴露的错误。
- **工具套件**：提供 `blender_scene_state`（运行时状态结构化输出）、多粒度预览、运动帧采样（`blender_motion_sheet_preview`）等工具，允许智能体直接读取引擎内部状态。
- **知识库**：自动从 Blender 源码提取的 API 文档库，由 `blender_docs` 工具按需查询，无需人工维护。

**算法流程**：
1. 规划者将文本提示编译为全局场景计划（JSON）。
2. 按固定阶段序列依次执行：每阶段由战术规划者生成阶段计划，编码者编写 bpy 脚本，验证器检查协议规则，审查者判断感知标准（渲染预览+引擎状态），规划者决定继续/重试/重新规划/终止。
3. 所有阶段通过后，渲染最终帧序列，并由最终审查者进行一次全局判断。

编辑模式：给定已有 .blend 文件和编辑指令，系统分解为编辑任务，无需重建静态元素。

## 3. 实验设计：数据集/场景、Benchmark、对比方法

**对比方法**：VIGA（Vision-as-Inverse-Graphics Agent），是目前唯一公开发布且支持动态场景生成的 Blender 多智能体系统，采用双智能体（生成器-验证器）循环。两者在 4DBuildBench 和 BlenderBench 上直接对比。

## 6. 主要结论与发现

1. **SimWorlds 在动态场景机制正确性上显著优于 VIGA**：整体 MPR 0.87 vs. 0.67，尤其在流体、刚体、粒子类别差距明显（流体 0.95 vs. 0.56）。
2. **视觉质量差距较小但仍占优**：VLM 分数 0.82 vs. 0.78，说明仅仅视觉隔离不足以区分机制真伪，而引擎状态审计能有效捕获伪造。
3. **编辑模式改进**：在 BlenderBench 上 PL、N-CLIP 均优于 VIGA，VLM 分数更高（3.41 vs. 2.76），表明分解式编辑优于全场景重审。
4. **消融验证**：验证器和阶段化管线各自防护不同维度，缺一不可（无验证器 SPR 下降 0.10，无阶段化 VLM 下降 0.11）。
5. **定性观察**：VIGA 在动态场景中常使用关键帧伪造物理效果（如角动量不合理、布料穿透），而 SimWorlds 通过求解器实现正确行为。

## 7. 优点

- **机制正确性首要**：首次将物理求解器的存在和正确配置作为生成的核心目标，而非仅视觉合理性。通过引擎状态审计解决了“看起来对但实际错”的关键失败模式。
- **双轨评估体系**：4DBuildBench 的引擎审计 + VLM 评判，可区分视觉质量与机制正确性，为后续研究提供更全面的评判标准。
- **模块化、可复用设计**：阶段化管线、场景协议、工具套件均独立于具体 LLM，可适配不同后端模型；知识库自动构建，无需人工维护。
- **编辑模式无缝集成**：同一框架无需修改即可处理从零生成和编辑指令，并在标准基准上取得改进。
- **透明性与可复现性**：代码、提示词、评估协议均开源，包含详细附录。

## 8. 不足与局限

- **感知判断仍依赖 LLM/VLM**：场景计划是否遵循提示词、美学构图等高级判断仍由 LLM 和 VLM 完成，这些模型不可靠，且无法由确定性规则替代。
- **文本输入限制**：当前仅支持文本提示，无法接收参考图像以约束布局和外观。作者指出未来可探索图像条件化。
- **运行时间长**：每场景 70–120 分钟，主要由 Blender 物理模拟烘焙时间主导，不适合实时或大规模批量生成。
- **实验覆盖面**：消融仅 15 场景，可能不完全反映全 50 场景的行为；每种子仅一次采样，未进行多轮统计。
- **基线比较的公平性**：迭代预算未完全对齐（VIGA 15 轮 vs. SimWorlds 每阶段有界重试），SPR 指标存在场景协议带来的先天偏差（作者已尽量补偿）。
- **应用限制**：作为程序化管线，输出质量受 LLM 编码能力和 Blender 版本限制；跨版本知识库可能需要更新；未评估在某些边缘案例（如极高复杂度跨物理交互）的表现。
